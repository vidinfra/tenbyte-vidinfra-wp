# Contributing to Vidinfra Player

Thank you for your interest in contributing to Vidinfra Player! This document provides guidelines for contributing to the plugin.

## Code of Conduct

Be respectful and considerate in your interactions with other contributors and users.

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in the issues
2. If not, create a new issue with:
   - Clear description of the problem
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - WordPress version
   - PHP version
   - Plugin version

### Suggesting Enhancements

1. Check if the enhancement has already been suggested
2. Create a new issue with:
   - Clear description of the enhancement
   - Use cases
   - Expected benefits

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature/fix
3. Write clean, documented code
4. Follow WordPress Coding Standards
5. Test your changes thoroughly
6. Submit a pull request with:
   - Clear description of changes
   - Reference to related issues
   - Screenshots (if applicable)

## Coding Standards

### PHP

- Follow [WordPress PHP Coding Standards](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/php/)
- Use meaningful variable and function names
- Add PHPDoc blocks for all functions
- Sanitize all inputs
- Escape all outputs
- Use WordPress core functions whenever possible

### JavaScript

- Follow [WordPress JavaScript Coding Standards](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/javascript/)
- Use modern ES5+ syntax when appropriate
- Comment complex logic
- Keep functions small and focused

### CSS

- Follow [WordPress CSS Coding Standards](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/css/)
- Use clear class names
- Add comments for complex selectors
- Maintain responsive design

## Testing

Before submitting:

1. Test on a clean WordPress installation
2. Test with common themes (Twenty Twenty-Four, etc.)
3. Test with Gutenberg and Classic Editor
4. Test all shortcode parameters
5. Test admin settings
6. Check for PHP errors and warnings
7. Verify security measures are in place

## Documentation

- Update readme.txt if adding features
- Update CHANGELOG.md with your changes
- Add inline comments for complex code
- Update README.md if necessary

## Questions?

Feel free to ask questions by creating an issue with the "question" label.

Thank you for contributing!
